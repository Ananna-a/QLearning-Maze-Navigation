# Python 强化学习迷宫程序设计：原理与流程理解

这份说明不是泛泛讲强化学习，而是对应你当前项目代码，帮助你理解：
1. 这个程序到底在学什么
2. 一次训练回合是怎么跑完的
3. 为什么会从随机走，逐步变成接近最优路径
4. 如何通过日志和现象判断“学会了”

---

## 1. 先用一句话理解这个作业

智能体在迷宫里不断试错，维护一张 Q 表（状态-动作价值表），
让“在某个位置做某个动作”这件事的分数越来越接近真实长期收益，
最后就会倾向于选择高分动作，从而走到终点并尽量少走弯路。

---

## 2. 三个模块分别在做什么

### 2.1 迷宫环境模块（maze_env.py）

你可以把环境理解成“游戏规则裁判”。

- 状态 State：智能体当前位置 `(x, y)`
- 动作 Action：`0/1/2/3` 分别代表上/下/左/右
- 奖励 Reward：
  - 到达终点：`+1`
  - 撞墙或越界：`-1`
  - 掉入陷阱：`-1`
  - 普通移动一步：`-0.01`

核心接口：
- `reset()`：回到起点，开始新回合
- `step(action)`：执行动作，返回 `(next_state, reward, done, info)`

重点理解：
- 你这里把撞墙设成了终止回合（done=True），所以“鲁莽动作”会被快速惩罚。
- 每步都有小负奖励，作用是逼智能体“更短路径”而不是“只要能到就行”。

### 2.2 强化学习大脑模块（rl_brain.py）

维护 Q 表，做两件事：

1) 选动作 `choose_action`
- 训练时：epsilon-greedy
  - 以 epsilon 概率随机探索
  - 否则选当前 Q 值最大的动作
- 测试/演示时：贪心选最大 Q

2) 学习更新 `learn`
- 公式：

$$
Q(s,a) \leftarrow Q(s,a) + \alpha\big[r + \gamma \max_{a'}Q(s',a') - Q(s,a)\big]
$$

可以拆成三步理解：
- 当前估计：`predict = Q(s,a)`
- 新目标值：`target = r + gamma * max Q(s',·)`（终止状态就只有 `r`）
- 用学习率 alpha 做“渐进修正”：向 target 靠近一点

### 2.3 主循环控制（main.py）

主程序负责把“环境 + 智能体 + 可视化”串起来。

- `train_agent(...)`：跑很多回合训练
- `evaluate_policy(...)`：训练后做纯贪心评估，算成功率
- `run_greedy_episode(...)`：可视化展示最终路径

你看到的“从乱走到会走”，本质就是：
- 早期 epsilon 大，随机探索多
- 后期 epsilon 衰减，更多利用已学到的高价值动作

## 3. 一次训练回合到底发生了什么

每个 episode 的流程可简化为：

1. `state = env.reset()` 回到起点
2. 循环最多 `max_steps` 步：
   - `action = agent.choose_action(state, training=True)`
   - `next_state, reward, done, _ = env.step(action)`
   - `agent.learn(state, action, reward, next_state, done)`
   - `state = next_state`
   - 若 `done=True`，本回合结束
3. 回合结束后 `agent.decay_epsilon()`
4. 记录日志（平均奖励、成功率、epsilon、Q表状态数）

这就是强化学习里最经典的“交互-反馈-更新”闭环。

## 4. 为什么它会“学会”

### 4.1 正负奖励塑造行为

- 到终点 +1 会提高沿途动作的 Q 值（通过 bootstrap 反向传导）
- 撞墙/陷阱 -1 会快速压低危险动作的 Q 值
- 每步 -0.01 让“绕远路”总回报更差

### 4.2 折扣因子 gamma 的作用

- gamma 越大，越重视长期回报
- 你设置在 0.95，属于“既看眼前也看未来”的平衡值

### 4.3 探索率 epsilon 的作用

- epsilon 大：先广泛试错，避免一开始陷入局部最优
- epsilon 小：后期稳定利用好策略

如果把学习想象成“修地图”：
- 初期地图几乎空白，只能乱探
- 每次奖励反馈都在给某条边打分
- 反复后，通向终点的边会越来越“亮”

## 5. 你应该如何观察训练是否正常

### 5.1 看日志

训练日志里常见指标：
- `avg_reward`：平均回报应整体上升（会波动）
- `success_rate`：成功率应逐渐提高
- `epsilon`：应逐步下降到设定下限附近
- `q_states`：访问过的状态数量逐渐增多并趋于稳定

### 5.2 看可视化

正常现象：
- 前期路径杂乱、经常失败
- 中期开始出现可重复的成功路线
- 后期路线更稳定，弯路减少

异常现象（可排查）：
- 一直失败：可能惩罚过重、探索不足或训练回合太少
- 成功率高但路径很长：步惩罚太小
- 学到后又退化：epsilon 衰减过慢或仍然探索过多

## 6. 参数怎么理解（先定性，再调参）

- 学习率 alpha（lr）
  - 大：学得快但可能不稳
  - 小：稳但慢
- 折扣 gamma
  - 大：更关注长期路径质量
  - 小：更短视
- epsilon / epsilon_decay / epsilon_min
  - 决定“探索->利用”切换节奏

入门建议（你现在这套就不错）：
- 先固定地图和奖励，不要一次改太多参数
- 每次只改 1 个参数，观察日志变化
- 用同一个随机种子做对比更公平

## 7. 一个实用的理解框架

当你看到任何一步时，脑中都问三个问题：

1. 当前我在哪个状态 s？
2. 我选了哪个动作 a，得到了什么即时奖励 r？
3. 这次反馈会让 Q(s,a) 上调还是下调？

只要这三个问题你都能答出来，Q-learning 的核心就已经理解了。
